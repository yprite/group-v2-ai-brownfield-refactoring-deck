AI Insight
~1분

중간 정리: 모델보다 작업장이 더 중요했습니다

좋은 모델도 context가 없으면 자신감 있게 틀릴 수 있습니다.

1
Harness beats prompt

프롬프트를 길게 쓰는 것보다, 실행과 검증이 반복되는 환경이 더 크게 작동했습니다.

agent harness
2
Context is a product

AI가 읽을 맥락은 덤프가 아니라 설계 대상입니다. 필요한 시점에 필요한 정보만 줘야 했습니다.

context engineering
3
Claims need artifacts

완료 선언은 신뢰하지 않았습니다. PR, 테스트, 측정 기록이 연결될 때만 완료로 취급했습니다.

artifact-backed claim
4
Fail closed by default
